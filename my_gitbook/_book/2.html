操作系统进程管理

操作系统进程管理

进程的基本概念

问题：程序在并发环境下运行结果不确定

方法：对运行过程进行制约

进程的定义

程序在某个数据集合上的一次运行活动（三个关键词）

进程的特征
  • 动态性（活动是动态的）
  • 并发性（进程和其他进程一起向前推进）
  • 异步性（进程按照自己的速度前进）
  • 独立性（进程是操作系统分配资源和调度CPU的基本单位）

注意这里不是分配CPU，因为在单CPU环境下，CPU只有一个，用分配不太合适。

我们可以类比人的一生

  • 动态性：人的一生由很多阶段，每个阶段都是不一样的
  • 并发性：每个人都在时间下慢慢推进
  • 异步性：每个人都按照符合自己的方式前进
进程与程序的区别

  • 动态与静态

    进程是程序在某个数据集合中的一次运行过程

    程序是指令的集合

  • 暂存和长存

    进程会有创建和消亡的过程

    程序在介质上长期保存

  • 程序与进程的对应关系

    一对一：一个程序运行一次

    一对多：一个程序运行多次

    多对一：程序运行时可以加载不同的程序

    多对多：并发执行不同的程序

进程分类

主要按照权限来分类，进程分为

  • 系统进程
  • 用户进程
进程的状态

  • 运行态：进程占用CPU

  • 等待态：进程具备运行条件但是没有占用CPU

  • 阻塞态：进程因为等待某项服务或信号不能运行的状态

三种状态的调度

就绪到运行：进程调度

运行到就绪：时间片到

运行到阻塞：请求服务，等待信号

阻塞到就绪：服务完成，信号到达

进程控制块（PCB）

进程控制块是一个数据结构

用来描述进程的状态、资源和与相关进程的关系。

  • PCB是进程的标志
进程=PCB+代码+数据

临界区和锁

临界资源：一次只允许一个进程独占访问（使用）的资源。

临界区：进程中访问临界资源的片段。

设计临界区访问的四个原则

  • 忙则等待
  • 空闲让进
  • 有限等待（不能一直饥饿）
  • 让权等待（如果某进程访问临界区失败，不能一直占着CPU，应该让出CPU给别人用）

锁

锁的引入是用来配合临界区。

当一个进程要进入临界区时，我们要查一下临界区是否上了锁:

  • 如果上了锁，进程就必须在临界区外面等着，直到锁解开
  • 如果没上锁，那进程就进入临界区然后再上锁；离开时，必须把锁解开

锁可以用一个bool变量来实现，0表示不可用（已经上锁），1表示可用（表示没有上锁)

同步和互斥的操作

进程的互斥

上面这个访问i的例子就是进程互斥的例子。

由于多个进程共享了独占性资源，必须协调顺序对资源进行存取；为了确保没有两个或以上的进程对独占性资源进行存取。这种关系叫做进程之间的互斥关系。

进程的同步

若干合作进程为了完成一个共同的任务，需要相互协调顺序；一个进程的运行必须需要另外一个进程完成，否则前面的进程只能等待。

司机和售票员之间的操作属于同步关系

假如司机的动作为：开车，行驶，停车

售票员的动作为：关门，售票，开门

司机和售票员的动作需要满足：

  • 关门->开车

  • 停车->开门

如果把司机和售票员当作两个进程，它们之间就是一种同步关系

实际上，进程的互斥也是一种特殊的同步关系

PV操作

P操作

  1. S --
  2. 如果S<0，把当前进程插入到q队列，然后阻塞本进程，
  3. 如果S>=0，该进程继续
P(){
    S -= 1;
    if(s < 0){
        Insert(Caller,q);
        Block(Caller);
        schedule();//转调度函数
    }
}

V操作

  1. S ++
  2. 本进程继续执行
  3. 如果S<=0，再从q队列中唤醒一个进程
V(){
    S += 1;
    if(S <= 0){
        Remove(q,pud);
        Wakeup(pid);
    }
}

PV操作解决互斥问题

PV操作可以解决互斥问题

P();//相当于上锁，因为P操作可以阻塞自身（相当于锁的自旋状态）

i = 1;
...
printf("%d",i);

V();//V操作会唤醒进程，相当于别的进程锁的访问成功

但是PV操作中S的初值设置一定要合理。

PV操作解决同步问题

PV操作解决同步问题的实质

当进程不满足条件时，需要让它立即停下来。

当进程满足条件式，经历让它马上继续。

因此我们在关键操作之前，可以进行一个P操作来阻塞不满足条件的进程

在关键操作之后，可以进行一个V操作来唤醒合作进程

WaitForXXXObject

WaitForMultipleObjects(参数)

函数功能：等待一组目标对象变成有信号的状态后返回

WaitForSingleObjects(参数)

函数功能：等待一个目标对象变成有信号的状态后返回

当线程结束的时候，线程会变成有信号的状态

程序分析

int S = 0;

A(){
    for(int i = 0;i < 100;i ++){
        S ++;
    }
}

int main(){
    //创建三个线程，三个线程使用的都是A函数
    Thread1 = createThread(A);
    Thread2 = createThread(A);
    Thread3 = createThread(A);
    WaitForMulitpleObject(参数);
    return 0;
}

当上面这个程序执行的时候，S的值并不一定是300（大概率不是300），因为S++这个语句不是原语，三个程序并发执行S++的时候，结果会不唯一。

如何解决？

  1. 定义临界区对象

  2. 将独占性资源放入临界区内
