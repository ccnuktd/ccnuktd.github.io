1.5 数据运算类指令

数据运算类指令

加法类指令

ADD DST,SRC ADD<--(DST) + (SRC) 修改6个标志寄存器

ADC DST,SRC 与ADD的区别为加的时候，需要带上进位位CF

INC DST 增1指令，修改除CF以外的5个标志寄存器

TF，IF和DF一般不在算数中使用

下面介绍一些这些寄存器的作用

  1. OF(overflow flag) 溢出标志位
  2. DF(direction flag) 方向标志位，一般用于处理字符串，DF=1按照递减处理，DF=0按照递增处理
  3. IF(interupt enable flag) 中断允许标志位，IF=1开中断，IF=0关中断
  4. TF(trap flag) 一般在单步跟踪的时候使用
  5. SF(sign flag) 符号标志位SF=0非负，SF=1负
  6. ZF(zero flag) 零标志位
  7. AF(auxiliary carry flag) 由于它在第四位，这个标志是辅助进位标志位（也可以处理借位）
  8. PF(parity flag) 奇偶，1的个数为偶数PF=1，奇数为0
  9. CF(carry flag) 进位标志位

加法指令例题复习P59例题3.2

减法类指令

SUB，SBB，DEC，NEG(求负) DST<--- 0 - (DST)

CMP DST, SRC (DEC) - (SRC) 根据结果改变6个状态寄存器，但是一般AF不用管，所以一般修改5个寄存器。

修改这5个寄存器，分别会有5种跳转

JZ,JNZ

JO,JNO

JS,JNS

JC,JNC

JP,JNP

除了修改5个寄存器，还有无符号数和带符号数比较产生的跳转

无符号数（高A低B）

带符号数（大G小L）

乘法类指令

MUL（无符号数乘法）和IMUL（带符号数乘法）

MUL/IMUL SRC（DST为隐含寻址，除此之外字节乘法还是字乘法看的是SRC的类型）

字节乘法(AL) SRC --> AX(88为16位)

字乘法 (AX) * SRC --> 高16位在DX，低16位在AX

我们可以通过CF和OF来判断高半部分有没有效

如果CF = 0 && OF = 0,那么高半部无效

如果CF = 1 && OF = 1,那么高半部有效

除法类指令

DIV（无符号数除法）和IDIV（带符号数除法）

字除法 (DX:AX)/SRC(16bit) 除数：AX 余数：DX

字节除法 (AX)/SRC(8bit) 除数：AL 余数：AH

字扩展指令

CBW 无操作数

CWD 无操作数

CBW将AL中的数扩展为AH:AL

CWD将AX中的数扩展为DX:AX

BCD数调整

分离BCD和组合BCD的区别

分离BCD，几位十进制就几个字节；组合BCD，几位十进制就几个半字节（4bit）

DAA 对AL中的组合BCD码进程加法调整

AAA 对AL中的分离BCD码进行加法调整

组合BCD数设计BCD码73 + 56：

MOV AL,73H
ADD AL,56H;这个时候AL=C9H
DAA;执行以后CF=1，AL=29H(+6修正法)

分离BCD数设计4+8：

MOV AL,04H
ADD AL,08H;这个时候AL=0CH
AAA;执行以后CF=1，结果为AX=0102H 
;得到这个结果需要用三部
;1.+6修正得到12H，AF=1
;2.AL高四位清0，即AL=02H
;3.AH = CF = AF = 1，即AX=0102H
;TIP:如果是用ASCII码加法，答案仍然一样，因此分离BCD码调整指令也叫ASCII码调整指令

DAS 对AL中的组合BCD码进程减法调整

AAS 对AL中的分离BCD码进行减法调整

修正的原则为减6修正原则，同时也会改变六个状态标志位

AAM 对AL中的分离BCD码进行乘法调整（先乘，再除10调整）【没有组合BCD的】

AAD 对AL中的分离BCD码进行除法调整（先调整，后除法）
