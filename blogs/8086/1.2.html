1.2 寻址方式

寻址方式

寻址方式——寻找操作数所在地或所在存储器单元地址的方式

寻址方式的种类

1.立即数寻址

在MOV AX,1234H这条指令中，操作数直接在指令中给出。

这里要注意，立即数只能做SRC。

MOV 1234H,AX这条语句有语法问题。

2.寄存器寻址

操作数在寄存器里。

MOV AX,BX，对于SRC使用寄存器寻址，对于DST也是用寄存器寻址。

因此寄存器中的数据可以相互传输。

但是寄存器的位必须相等。

MOV AX,CL这条语句中SRC和DST位数不同，语法错误。

注意：

  1. 对于MOV [2000H],56H，这条语句语法错误。因为SRC和DST都无法确定类型。所以我们必须使用PTR去强制确定类型：MOV WORD PTR [2000H],56H。这个时候由于DST的类型明确，SRC会默认位DST的类型。

  2. 立即数以及立即数寻址无法为段寄存器(DS,SS,ES)赋值。

    例如：MOV SS,1500H(X)

    可以通过一个中介来实现功能。

    MOV AX,1500H
MOV SS,AX

  3. CS和IP无法做DST，用户无法改变CS和IP。

  4. 两存储单元之间无法直接进行操作。

3.存储器寻址——要寻找的OPR(操作数)在某存储单元中，OPR的EA(effective address)可以由以下5中寻址方式求得

  • 直接寻址
  • 寄存器间接寻址（可用寄存器：BX,SI,DI）
  • 寄存器相对寻址（可用寄存器：BX,BP,SI,DI + 8位或16位段内偏移）
  • 基址，变址寻址（基址：BX,BP 变址：SI,DI）
  • 基址，变址且间接寻址

这一些寻址方式怎么用呢？

  • 直接寻址

    操作数所在单元的EA直接在指令中给出。

    MOV AL,[2000H]，这条语句去找了段地址（默认为DS），段内偏移为2000H的地址，取出这个地址的值给AL（由于AL是8位的，取出的为字型数据）。

  • 寄存器间接寻址

    能使用寄存器间接寻址的寄存器有三种（BX,SI,DI），默认段为DS段。

    例如：MOV BX,offset dat1这个地方的offset是立即数寻址，BX中存储dat1的段内偏移。

    MOV AL,[BX]把BX所在段内偏移中的值取一个字节放入AL中。

  • 寄存器相对寻址

    寄存器相对寻址由BX,BP,SI,DI提供基地址（其中，BX,SI,DI的段默认为DS段；BP的段默认为SS段），再加上一个8位或16位的disp（相对位移量）

这里要注意：寄存器间接寻址和寄存器相对寻址

MOV AL,[BX]

MOV [BX],AL

第一条指令为寄存器间接寻址，第二条指令为寄存器相对寻址（只不过偏移是0）

除此之外如果给出的相对位移量是一个变量的话，段地址以变量的段地址为准

如果要在DAT1底下3个字节写入AL的内容

MOV BX,OFFSET DAT
MOV [BX] + 3,AL;MOV [BX + 3],AL;MOV 3 + [BX],AL
;或者下面这个也可以
MOV DAT1 + 3,AL

还有一种MOV dat[BX],AL，这个和数组非常相似，dat是一个地址（相当于变量的段内偏移，接着加上BX内的值）

有一个关于[]很好的理解：[BX]为BX寄存器内的值，BX就是BX寄存器

MOV BX,1000H是把16进制的1000写到BX寄存器里面；MOV [BX],1000H是把16进制的1000写到以BX内的值为偏移地址的内存单元中

MOV指令的重大误区：

MOV指令的操作数都是**地址**：

下面用一个例题来说明：

;已经执行
PUSH AX
PUSH BX
PUSH CX
;在不破坏SP寄存器（不修改）的情况下，用汇编代码将AX的内容写到DX中

;解答
;这里要注意SP随着栈中内容的增多而减少（**栈向低字节扩展**）
;SP的指向如下图


MOV BX,SP;BX的作用就是代替SP来破坏，而且它的段在堆栈段，正好和SP是一个段
;如果BX也要保护的话，可以PUSH BX
MOV DX,[BX] + 4

这个地方不要写成MOV DX,[[BX] + 4]

[[BX] + 4]的含义是取出以BX内的值 + 4为地址的单元的值，这个地方取出的是所在单元的内容（也就是所谓的值），MOV指令的操作数都是地址。在执行MOV指令的时候，CPU会根据MOV的操作数进行一次寻址，如果写成MOV DX,[[BX] + 4]，就多了一次寻址，最后的写道DX中的东西会是，以AX内的值为地址再寻址的地址内的值。

  • 基址变址寻址

基址寄存器为BX,BP，变址寄存器为SI,DI。（BX在DS段，BP在SS段）

  • 基址变址且相对寻址

    和前面一样

4.隐含寻址

PUSH AX指令中无目的操作数，但是CPU知道默认的目的操作数的位置(目的操作数为隐含寻址)。

CPU实际执行的是：

;下面用伪代码写
(SP) <- (SP) - 2
SS:(SP) <- (AX)
